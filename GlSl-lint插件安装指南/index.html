GlSl-lint插件安装跳坑

发表于 2023-11-08

VS Code 在安装完GlSl lint插件后*.glsl文件依然无法通过 lint 规则，同时提示报错信息 “Failed to spawn ‘glslangValidator‘ binary. Error: spawn glslangValidator ENOENT”

一号坑

网上有通过关闭 security.workspace.trust 来解决，尝试不起作用后寻找其他办法。

解决方法一

出现“Failed to spawn ‘glslangValidator’ binary. Error: spawn glslangValidator ENOENT”的原因是glsl lint找不到glslangValidator在哪。

这里给出glslang的Releases，可以自行下载对应版本。（macOS 选择 osx）

以下为 macOS 操作步骤

下载完后按照以下操作步骤：

  1. 解压下载文件，分别有bin,include,lib三个文件夹
  2. 打开/usr/local/将include和lib文件夹复制进去
  3. 将bin文件夹下glslangValidator复制到/usr/local/bin/文件夹下，如果没有bin文件夹可以创建一个
  4. 打开vs code设置，定位到glsl lint拓展，设置”Glsllint: Glslang Validator Path”内容为 /usr/local/bin/glslangValidator
  5. 重启vs code，查看插件是否成功运行

二号坑

插件依旧不生效，首先查看“隐私与安全性” => “安全性”是否有权限允许选项，也可以手动允许一次glslangValidator，系统会有安全提示，允许就可以了。
一些设置完毕，记得重启vs code。

三号坑

错误提示 “GLSL Lint: Wrong parameters when starting glslangValidator”
如果使用单文件 *.glsl, 来引入glsl，则需要调整glsl文件后缀，比如顶点着色器后缀.vert，片元着色器后缀.frag。
这里是相关映射 Link

题外话

如果使用webpack构建webgl之类，可能无法识别glsl相关需要文件，则需要相关配置rules，如果是vite在import url后加上?raw即可。

windows用户也是类似操作，解压完后，设置”Glsllint: Glslang Validator Path”内容为glslangValidator地址就可以了。
